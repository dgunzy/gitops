# Flux Operator GitOps Model (No Helm Runtime Path)

This cluster now uses Flux Operator APIs as the control plane for Flux lifecycle,
while Flux itself reconciles the operator manifests from Git.

## Target State

- `FluxInstance` manages Flux install and Flux upgrades (`clusters/k0s-cluster-1/flux-instance.yaml`).
- Flux root sync (`flux-system` Kustomization generated by `FluxInstance.spec.sync`) reconciles:
  - `clusters/k0s-cluster-1/flux-instance.yaml`
  - `clusters/k0s-cluster-1/flux-sync.yaml`
  - `clusters/k0s-cluster-1/flux-operator` (operator source + kustomization)
- Operator manifests are applied by Flux from upstream tagged source:
  - `GitRepository/flux-operator`
  - `Kustomization/flux-operator` with `path: ./config/default`
- Platform/app dependency chain is explicit:
  - `platform-controllers` -> `platform-configs` -> `apps`

## Why This Pattern

- No Helm required for day-2 operator management.
- Flux controller upgrades are automatic via `FluxInstance.spec.distribution.version` semver.
- Flux Operator upgrades are automatic via `GitRepository.spec.ref.semver`.
- All steady-state changes flow through Git and Flux reconciliation.

## Migration From Existing Helm-Installed Operator

1. Commit/push this repo layout.
2. Keep the current operator running; do not uninstall it yet.
3. Ensure `FluxInstance` is applied:
   - `kubectl apply -f clusters/k0s-cluster-1/flux-instance.yaml`
4. Wait for reconciliation:
   - `kubectl -n flux-system wait fluxinstance/flux --for=condition=Ready --timeout=10m`
   - `kubectl -n flux-system wait kustomization/flux-operator --for=condition=Ready --timeout=10m`
5. Verify control-plane health:
   - `flux get all -n flux-system`
   - `kubectl -n flux-system get fluxreport flux -o yaml`
6. Stop using Helm as the source of truth for `flux-operator`.

## Bootstrap for New Cluster (No Helm)

1. Seed operator once:
   - `kubectl apply -k "github.com/controlplaneio-fluxcd/flux-operator/config/default?ref=v0.42.1"`
2. Apply `FluxInstance`:
   - `kubectl apply -f clusters/k0s-cluster-1/flux-instance.yaml`
3. Flux takes over and reconciles the full graph from Git.

## Upgrade Policy Knobs

- Flux controllers:
  - `clusters/k0s-cluster-1/flux-instance.yaml`
  - `spec.distribution.version` (currently `2.x`)
- Flux Operator:
  - `clusters/k0s-cluster-1/flux-operator/source.yaml`
  - `spec.ref.semver` (currently `0.42.x`)

Use patch-only ranges for conservative rollout, broaden to minor ranges when desired.
